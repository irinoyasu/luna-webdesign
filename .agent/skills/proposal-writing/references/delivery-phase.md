# Delivery Phase Reference

## Purpose

Package, submit, and actively sell the proposal. A great proposal delivered poorly is a wasted effort. This phase covers everything from final packaging to pitch preparation to follow-up strategy.

**Key principle:** The proposal is the beginning of a conversation, not the end. Delivery strategy should maximize the chance that the proposal leads to a productive dialogue, not just a "received, thank you" email.

> **📖 Cross-reference:** For persuasive writing patterns applicable when preparing pitch scripts and Q&A responses, see [persuasive-writing-guide.md](../assets/persuasive-writing-guide.md).

### 2. Prepare Pitch Materials

Prepare materials for presenting the proposal in person or virtually. Pitches bring the proposal to life through human connection and real-time engagement.

**Instructions:**

1. **Create Pitch Versions:**

   **10-Minute Presentation:**
   | Time | Section | Content | Visual |
   |---|---|---|---|
   | 0:00-1:00 | Hook | Self-introduction + Customer challenge + stakes | Problem visual |
   | 1:00-3:00 | Understanding | Customer's situation, your research | Customer context |
   | 3:00-6:00 | Solution | Your approach + win themes | Solution overview |
   | 6:00-8:00 | Proof | Evidence, case studies, team | Results + credentials |
   | 8:00-9:00 | Investment | Pricing, ROI, risk mitigation | Key metrics |
   | 9:00-10:00 | Close | Commitment, next steps, ask | Call to action |

   Slide rules:
   - Maximum 10 slides for 10 minutes
   - One key idea per slide
   - Minimal text (headlines only)
   - High-impact visuals

2. **Prepare Q&A Responses:**
   Anticipate the top 10-15 questions and prepare confident responses:

   | Category | Question | Key Message | Supporting Data | Pitfall to Avoid |
   |---|---|---|---|---|
   | Technical | | | | |
   | Team | | | | |
   | Pricing | | | | |
   | Risk | | | | |
   | Competition | | | | |

   For difficult questions, use the **bridge technique:**
   - Acknowledge the question briefly
   - Bridge to your key message
   - Reinforce a win theme

3. **For Bilingual Proposals (JP/EN):**
   Create pitch scripts in both languages. Key messaging should be culturally adapted, not just translated.

**Outputs:**
- Pitch script (10-minute, starting with a self-introduction)
- Q&A preparation document
- Slide deck outline (if presentation needed)

### 3. Plan Follow-Up

Design the engagement strategy for after submission. The follow-up is often what converts a good proposal into a signed contract.

**Instructions:**

1. **Follow-Up Timeline:**
   | Day | Action | Channel | Goal |
   |---|---|---|---|
   | Day 0 | Submit proposal | Email/portal | Confirm receipt |
   | Day 1-2 | Confirmation follow-up | Phone/email | Verify received, offer to answer questions |
   | Day 7 | Check-in | Email | "Any questions about the proposal?" |
   | Day 14 | Value-add touchpoint | Email | Share relevant article, case study, or insight |
   | Day 21 | Meeting request | Phone | Offer in-person/virtual presentation |
   | Day 30+ | Decision follow-up | Phone | Inquire about timeline, offer to address concerns |

2. **Objection Handling Preparation:**
   Based on the Review phase's objection list, prepare responses for the most likely push-back:
   | Likely Objection | Response Strategy | Supporting Material |
   |---|---|---|
   | | | |

3. **Win/Loss Debrief Plan:**
   Regardless of outcome, plan to learn from this opportunity:
   - If won: What worked? Document for future proposals
   - If lost: Request debrief from customer. What would have changed the decision?
   - Key lessons to capture either way

4. **Relationship Maintenance (if not immediately won):**
   - Add customer contacts to appropriate nurture sequence
   - Schedule periodic touchpoints (quarterly check-in, industry news sharing)
   - Monitor customer's public announcements for future opportunities

**Outputs:**
- Follow-up plan with timeline and actions
- Objection response guide
- Win/loss debrief plan
