# Review Phase Reference

## Purpose

Evaluate the proposal through the customer's eyes. Find and fix weaknesses before the customer finds them. This phase transforms a good proposal into a winning one through structured, honest critique.

**Key principle:** A review is only valuable if it's willing to find problems. "Looks good" is not a review output. Every review must produce specific, actionable improvements.

## Sub-Task Index

| # | Sub-Task | Output File | Description |
|---|---|---|---|
| 1 | Compliance & Accuracy Check | `review/compliance-check.md` | Verify all requirements are addressed and facts are accurate |
| 2 | Persuasiveness Review | `review/persuasiveness-review.md` | Score from the customer's perspective — would this win? |
| 3 | Final Quality Gate | `review/quality-gate.md` | Sign-off decision with conditions |

---

## Review Framework

Reviews follow the Shipley color team model, adapted for efficiency:

| Stage | Focus | When | Key Question |
|---|---|---|---|
| **Compliance Check** (Pink Team idea) | Completeness, accuracy, structure | After first complete draft | "Is everything there and correct?" |
| **Persuasiveness Review** (Red Team idea) | Win probability, customer impact | After revisions | "Would this convince the customer?" |
| **Quality Gate** (Gold Team idea) | Executive sign-off, final readiness | Before delivery | "Are we confident enough to submit?" |

## Detailed Sub-Task Instructions

### 1. Compliance & Accuracy Check

Ensure the proposal is complete, accurate, and addresses all requirements. This is the foundation — persuasiveness doesn't matter if the proposal is wrong or incomplete.

**Instructions:**

1. **Requirements Coverage (for RFP responses):**
   | Req # | Requirement | Addressed? | Section/Page | Notes |
   |---|---|---|---|---|
   | | | ☐ Full ☐ Partial ☐ No | | |

   Compliance score: ___% fully addressed

2. **Pain Point Coverage (for unsolicited proposals):**
   | Pain Point | Addressed in Proposal? | Section | Strength |
   |---|---|---|---|
   | | ☐ Yes ☐ Partial ☐ No | | Strong / Adequate / Weak |

3. **Factual Accuracy:**
   - [ ] All numbers verified (pricing, timelines, metrics)
   - [ ] Customer name and details correct throughout
   - [ ] Case study facts accurate and currently valid
   - [ ] Market data cited with sources
   - [ ] No claims that cannot be substantiated

4. **Internal Consistency:**
   - [ ] Pricing matches across pitch versions, solution, and pricing sections
   - [ ] Timeline is consistent (milestone dates, durations)
   - [ ] Team member names and roles match across sections
   - [ ] Terminology used consistently

5. **Structural Quality:**
   - [ ] All required sections present
   - [ ] Table of contents matches actual sections
   - [ ] Headers and numbering correct
   - [ ] Cross-references valid

### 2. Persuasiveness Review

### 3. Final Quality Gate

Executive-level final check. Focus on readiness to submit and confidence level.

**Instructions:**

1. **Readiness Checklist:**
   - [ ] All Compliance Check issues resolved
   - [ ] All Persuasiveness Review improvements implemented
   - [ ] Pitch versions are compelling and complete
   - [ ] Pricing is competitive and well-justified
   - [ ] All claims have proof
   - [ ] Document is formatted and professional
   - [ ] All appendices and supporting materials ready

2. **Confidence Assessment:**
   | Question | Answer |
   |---|---|
   | Are we confident this addresses the customer's top priorities? | |
   | Does our pricing represent genuine value? | |
   | Can we deliver what we've promised? | |
   | Would we bet our reputation on this proposal? | |

3. **Decision:**
   - **APPROVE** — Ready to submit/deliver
   - **APPROVE WITH CONDITIONS** — Submit after addressing specific items
   - **RETURN FOR REVISION** — Significant improvements needed before submission

4. **Remaining Risks:**
   | Risk | Status | Acceptable? |
   |---|---|---|
   | | | ☐ Yes ☐ No |

**Outputs:**
- Quality gate decision with conditions
- Final risk acceptance
