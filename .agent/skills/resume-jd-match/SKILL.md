# Resume JD Match — JD最適化履歴書ジェネレーター

洗練された、JDにマッチしたHTML履歴書とPDFを生成します。2フェーズのワークフロー：プロファイル収集 → JDマッチング → 生成。

## フェーズ 1: プロファイル収集

初回実行時（`resume-profile.md`が存在しない場合）、ユーザー情報を収集します。セクションごとに1つずつ質問し、確認してから次へ進みます。内容は`resume-profile.md`に保存されます。

**最初に好みを設定します：**
1. **アクセントカラー** — 16進数または色名。デフォルトは `#6b4c9a`
2. **言語** — 日本語 または English
3. **対象地域** — 日本国内（生年月日を含める）または海外（除外する）

### セクション（各セクションの構造化データ）
1. 基本情報 — 氏名、電話番号、メールアドレス、生年月日、現住所、LinkedIn
2. 学歴 — 学校名、学位、専攻、期間、受賞歴・成績
3. 職務経歴 — 各ロール：会社名、役職、期間、弾丸ポイント3-6項目（数値を活用！）
4. サイドプロジェクト — 各プロジェクト：名称、タイプ、ステータス、役割、特徴/指標、使用技術
5. スキル — 専門領域 ＋ 技術ツール ＋ 言語、グループ化して記載
6. 資格・表彰

2回目以降の実行：プロファイルを読み込み、「更新事項はありますか？」と確認します。

## フェーズ 2: JDマッチングと生成

### 2.1 JDの収集
貼り付けられたテキスト、ファイル、またはURLを受け付けます。解析後、続行前にユーザーに内容を確認します：
- 職種名、会社名、主要要件（3-5項目）、歓迎条件

### 2.2 マッチング分析（ユーザーに表示し、確認を待つ）
```
🎯 ターゲット: [職種名] @ [会社名]
📊 マッチ度: [高/中/低]
✅ 強み・適合点: ...
⚠️ ギャップ・不足点: ...
📝 戦略: [強調する点 / 言い換える点 / 控えめにする点]
```

### 2.3 HTML履歴書の生成

インラインCSSを含む単一の自己完結型HTML。詳細は [references/html-template-guide.md](references/html-template-guide.md) を参照。

**コンテンツルール:**
- 上部のハイライトグリッドに3つのコアアドバンテージを記載 → JDの最優先事項に対応
- 職務経歴はJDへの関連度順に並べる
- JDのキーワードを自然に取り入れ、不自然な詰め込みは避ける
- 可能な限りすべての項目で実績を数値化する
- スキルセクションはJDで要求されているスキルを優先する（tag-highlight クラスを使用）
- A4サイズで最大1-2ページ

### 2.4 PDFエクスポート

```powershell
powershell -File scripts/export-pdf.ps1 -HtmlPath "$htmlPath" -OutputPath "$pdfPath"
```

エクスポートに失敗した場合：ユーザーにHTMLをブラウザで開き、Ctrl+P（印刷）でPDF保存するよう案内します。

### 2.5 納品

`resumes/` フォルダに保存： `[氏名]-[職種].html` ＋ `[氏名]-[職種].pdf`

送信前チェックリスト：
- 連絡先が正しいか
- 日付が正確か
- 会社名やプロジェクト名にタイポがないか
- 実績が誇張されていないか
- PDFのフォーマットが崩れていないか
