# Strategy Phase Reference

## Purpose

Transform research intelligence into a compelling win strategy. Define what makes you the right choice, how to differentiate from alternatives, and craft messaging that resonates with each audience in the decision-making unit.

**Key principle:** Strategy is about *choices*. You cannot be everything to everyone. Choose which pain points to lead with, which differentiators to emphasize, and which audience to prioritize.

> **📖 Cross-reference:** For persuasive writing patterns (BLUF, Proof Triangle, Ghost Competitor Technique) applicable when crafting win themes and value propositions, see [persuasive-writing-guide.md](../assets/persuasive-writing-guide.md).

## Detailed Sub-Task Instructions

### 1. Develop Win Themes

Win themes are concise, customer-focused statements that answer: **"Why should the customer choose us?"** They anchor the entire proposal narrative — every section should reinforce at least one win theme.

**Inputs:**
- Customer pain points and priorities (from Research)
- Proof point inventory (from Research)
- Your capabilities and differentiators

**Instructions:**

1. **Draft 3-4 Win Theme Statements:**
   Use this formula for each:

   > **[Customer need/pain point]** + **[Your differentiator]** + **[Quantified outcome/benefit]**

   Example: "Your plant's 200+ hours of annual cleaning downtime drops to under 20 hours with our tool-free disassembly design, proven across 100+ installations in pharmaceutical manufacturing."

   Rules for effective win themes:
   - ≤25 words each
   - Customer-focused (about them, not you)
   - Specific and quantified (avoid vague claims like "best-in-class")
   - Differentiated — if a competitor could say the same thing, it's not a win theme
   - Aligned with what matters most to the decision-maker

2. **Attach 2-3 Proof Points Per Theme:**
   | Win Theme | Proof Point | Type | Source |
   |---|---|---|---|
   | | | Case study / Metric / Certification / Award / Testimonial | |

3. **Stress Test Against Competitors:**
   | Win Theme | Your Claim | Competitor's Likely Counter | Your Rebuttal |
   |---|---|---|---|
   | | | | |

4. **Plan Theme Deployment:**
   Note where each theme should appear most prominently:
   | Win Theme | Pitch Versions | Solution Section | Credentials | Pricing/ROI | Presentation |
   |---|---|---|---|---|---|
   | Theme 1 | ✓ (lead) | ✓ | | ✓ | ✓ |

**Outputs:**
- Win theme guide with statements, proof points, stress tests
- Theme deployment matrix

### 2. Craft Value Proposition

Articulate why your proposal delivers the best value — tailored to each person in the decision-making unit. Different readers have different concerns.

**Inputs:**
- Win themes (from sub-task 1)
- Decision-making unit map (from Research)
- Customer pain points

**Instructions:**

1. **Segment Your Audience:**
   | Audience | Primary Question | What Convinces Them |
   |---|---|---|
   | Executive sponsor | "Does this advance our strategy?" | Strategic alignment, vision, ROI |
   | Technical evaluator | "Can they actually do this?" | Evidence, methodology, credentials |
   | Financial evaluator | "Is it worth the money?" | ROI, cost comparison, risk mitigation |
   | End user / operator | "Will this make my work better?" | Usability, support, practical benefits |
   | Procurement / legal | "Is it safe to choose them?" | Compliance, terms, references |

2. **Define 3-4 Value Pillars:**
   Group your differentiators into clear categories:
   | Value Pillar | Description | Key Metric |
   |---|---|---|
   | e.g., Risk Reduction | | |
   | e.g., Cost Optimization | | |
   | e.g., Speed to Value | | |
   | e.g., Domain Expertise | | |

3. **Create Audience-Specific Messaging:**
   For each audience × value pillar, write a 1-2 sentence message:
   | Audience | Value Pillar | Message | Where Used |
   |---|---|---|---|
   | Executive | Risk Reduction | "..." | Pitch Versions |
   | Technical | Domain Expertise | "..." | Solution Section |

4. **Address Key Risks Per Audience:**
   | Audience | Their Key Fear | Your Mitigation | Evidence |
   |---|---|---|---|
   | | | | |

**Outputs:**
- Value proposition playbook with audience-specific messaging
- Risk mitigation table per audience

### 3. Define Competitive Positioning

Position your proposal against alternatives — without naming competitors directly. Use the **ghost competitor technique**: address competitor weaknesses indirectly by contrasting approaches, not companies.

**Instructions:**

1. **Map Alternatives:**
   | Alternative | Brief Description | Key Strength | Key Weakness |
   |---|---|---|---|
   | Do nothing (status quo) | | | |
   | Competitor A (type) | | | |
   | Competitor B (type) | | | |
   | In-house solution | | | |

2. **Create Ghost Competitor Messaging:**
   | Competitor Weakness | Ghost Statement (never name them) |
   |---|---|
   | Requires costly on-site work | "Unlike approaches that require costly on-site customization, our solution deploys remotely in 48 hours." |
   | Lacks domain expertise | "Where generalist agencies struggle to communicate technical nuances, our team's [specific credential] ensures..." |

3. **Build "Why Not" Answers:**
   Anticipate objections about why they should NOT choose an alternative:
   | "Why not just...?" | Your Response |
   |---|---|
   | "...keep doing what we're doing?" | |
   | "...use a cheaper alternative?" | |
   | "...do it in-house?" | |
   | "...wait until next year?" | |

**Outputs:**
- Competitive positioning map
- Ghost competitor messaging library
- "Why not" response guide
